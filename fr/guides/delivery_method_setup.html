Définir des méthodes de livraison¶

Avant de pouvoir générer nos premières étiquettes nous avons besoin de définir une ou plusieurs Méthodes de livraison du type PostLogistics.

Il existe de nombreuses combinaison d’option disponibles que nous n’énumérerons pas ici. Pour en obtenir une liste exhaustive référez vous en au Manuel Webservice “Code à barres”.

Créer une méthode de livraison¶

Allez dans Inventaire/Configuration/Livraison/Méthodes de livraison et créer un nouvelle Méthode de livraison :

Voici les étapes pour configurer votre Méthode de livraison:

  1. Donnez un nom à votre méthode de livraison

  2. Choisissez PostLogistics dans le champs Type

  3. Choisissez un Groupe de service. Référer vous au Manuel Webservice “Code à barres” pour savoir quel service est inclut dans chaque groupe. Ici, vous allez probablement vouloir sélectionner le groupe de service “Colis”.

  4. Optionnellement sélectionnez une licence. Si vous avez effectuer l’étape avec le bouton Assigner les licences PostLogistics à un groupe de service un essai de détection sera effectué pour savoir quelle licence utiliser.

  5. Définissez la Société de transport en sélectionnant PostLogistics

Sauvez (Cette étape est important car elle va rafraîchir les options disponible)

Service Basique¶

Ajouter un service basique en ajoutant une option.

  1. Cochez obligatoire.

  2. Choisissez un service de base

    Note

    Seulement les options de base du groupe de service sélectionné seront disponible.

    Si votre option n’est pas la essayez de sélectionner un autre groupe de service, par exemple si vous cherchez Encombrant “Eclair” celui ci est dans le groupe de service Exprès / Coursier.

  3. Sauvegarder & fermer

  4. Sauvegarder la Méthode de livraison

La méthode de livraison est prête à l’emploi. Cependant, vous pourriez vouloir ajouter quelques services et/ou des instructions de livraison.

Services optionnels et instructions de livraison¶

Assurez vous que vous avez sauvegardé la Méthode de livraison lors de la configuration du Service de base (Nous devons rafraîchir la liste d’options, une fois rafraîchie les instructions de livraison seront disponible).

Maintenant vous pouvez définir autant d’autre option et d’instruction de livraison dont vous avez besoin.

Il y a 3 types d’options:

  • Obligatoire : Chaque bon de livraison avec cette méthode de livraison aura cette option. C’est une bonne pratique de définir le service de base (ex. ECO) en obligatoire car vous aurez toujours besoin d’un service de base lors de la génération d’étiquette.

  • Appliqué par défaut: Cette option sera ajouté sur le bon de livraison. Elle pourra cependant être retirée par le logisticien.

  • Optionnelle: Celle ci ne sera pas ajoutée sur le bon de livraison. Elle deviendra un option autorisé pouvant être ajouter par la suite sur le bon de livraison.

Note

Les options seront copiées depuis la Méthode de livraison vers le Bon de livraison lors de la sélection de la Méthode de livraison sur le formulaire du Bon de livraison.

Définir des options non obligatoires donne la liberté au logisticien de faire quelques changement pour configurer les étiquettes. Et ainsi lors de la génération des étiquette de livraison, les options des bon de livraison seront utilisées.

Ceci peut être utile pour gérer manuellement les cas exceptionnels sur les bons de livraisons. Ex. Vous pouvez ajouter une instruction de livraison en optionnel pour les rares cas ou le client veut être appelé par téléphone lorsque le colis arrive.

Format de sortie¶

Vous pouvez aussi ajouter des option pour surcharger le format de sortie. Autrement, l’étiquette résultante sera générée à partir des valeurs par défaut définies dans la configuration PostLogistics.
